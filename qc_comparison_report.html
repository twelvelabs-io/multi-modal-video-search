Key Findings

What We Learned

  • ●Combined is recommended. Cosine handles shot-level structural edits (added, removed, reordered content) while L2 adds sensitivity to color grading, luma adjustments, and text overlay changes that cosine normalizes away.
  • ●Embeddings capture subtle differences. L2 distributions clearly separate modified segments from clean ones. PH_Test2_I-008 (3% luma + text changes): 17 segments flagged by combined vs 0 by cosine alone.
  • ●Standards conversion (50i) is transparent. PH_Test2_H-011 shows 0 flags across all methods. Embeddings are field-order invariant.
  • ●2% time compression/extension is below detection threshold. Sub-5% timing changes don't alter per-segment embedding content enough to flag. VMAF or frame-count analysis needed for these cases.
  • ●Performance is real-time. Positional alignment (same segment count): ~25ms. Greedy alignment (different counts): ~1.3s. Combined adds ~5ms overhead.

Methodology

How It Works

Each video is segmented into 1-second clips and embedded with AWS Bedrock Marengo 3.0 (512 dimensions, visual + audio + transcription modalities). Segments are aligned positionally when counts match, or via greedy best-match when counts differ. Each aligned pair is scored by cosine similarity (directional match) and L2 distance (magnitude match). Differences are automatically classified based on which metric flags, modality patterns, and time displacement. VMAF pixel-level scoring is available per segment through the app’s QC analysis endpoint for secondary validation.

Thresholds: Cosine < 0.95 = flagged. L2 < 0.85 = flagged. Combined = OR of both. Classifications: removed_content, added_content, significant_change (both fail), color_luma_change (L2-only), compression_artifact (minor), shot_reorder (time displacement > 5s).
